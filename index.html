When Future Covariates Are Unavailable, Replay Last Year’s Season and Rescale It

A deployment-time approximation strategy for missing traffic and weather inputs

View Repository Explore Method

Overview

This project highlights a specific deployment problem inside the Nairobi crash prediction pipeline: some of the strongest covariates, especially Uber Movement traffic speeds and weather, were not available for the future prediction period.

The test window covered July to December 2019, but real future values for those variables did not exist at scoring time. Instead of dropping them or freezing the last observed values, the pipeline used a more structured approximation.

The core tactic was simple: reuse the same seasonal window from the prior year, then rescale it using the recent year-over-year level ratio.

Architecture

The Core Idea

The goal was not to forecast exact future traffic speed values.

The goal was to preserve three things:

  • the shape of the relevant seasonal window
  • the relative structure across segments and time bins
  • the level shift implied by recent year-over-year change

Seasonal replay × recent level adjustment is often more useful than stale carry-forward.

The approximation logic

Bad option: drop the feature

Safe, but expensive. You lose a variable that clearly carries operational signal.

Weak option: freeze the last value

Keeps the feature but destroys seasonality and relative temporal movement.

Chosen option: replay + rescale

Reuse the prior-year seasonal window and shift it using the recent year-over-year ratio.

Selected Code Snippet

uber_2018_h2 <- uber_by[year == 2018 & month %in% 7:12]
uber_2019_h1 <- uber_by[year == 2019 & month %in% 1:6]
uber_2018_h1 <- uber_by[year == 2018 & month %in% 1:6]

trend_ratio <- mean(uber_2019_h1$speed, na.rm = TRUE) /
    mean(uber_2018_h1$speed, na.rm = TRUE)

uber_future <- copy(uber_2018_h2)
uber_future[, year := 2019L]
uber_future[, speed := speed * trend_ratio]

In practice, this means:

  • take July–December 2018 as the structural template
  • measure how Jan–June 2019 differs from Jan–June 2018
  • apply that ratio to the replayed second-half seasonal pattern

Why this was useful

  • kept informative covariates in the deployed model
  • preserved seasonality better than last-observation carry-forward
  • introduced a simple level correction without pretending exact future knowledge
  • fit naturally with traffic and weather variables that have recurring annual structure

Where this pattern generalizes

  • traffic and mobility modeling
  • weather-dependent operational prediction
  • economic indicators with delayed publication
  • demand and supply systems with future covariate gaps

The reusable idea is not “future features may be missing.” The reusable idea is a specific approximation strategy: reuse prior seasonal geometry, then shift the level using recent trend.

Key Takeaway

When exact future covariates are unavailable, a seasonal replay with year-over-year rescaling can preserve enough structure to keep the feature useful.
